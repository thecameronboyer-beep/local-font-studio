App Shell and Navigation
========================

Main files
----------
- src\App.tsx
- src\main.tsx
- src\pwa.ts
- src\styles.css

What this category covers
-------------------------
This category covers the React app shell: startup, project loading, active font selection, glyph selection, fullscreen panels, saved images, app themes, and the controlled callbacks that let child components mutate the project.

Mounting
--------
src\main.tsx imports React, ReactDOM, App, styles.css, and registerServiceWorker. The service worker only registers in production.

App.tsx state
-------------
- studioData: the full FontStudioData project loaded from localStorage.
- selectedCharacter: active glyph key, default "A".
- editorFullScreen: shows the glyph editor overlay.
- gridFullScreen: shows the alphabet overlay.
- savedImagesOpen: shows the saved PNG gallery.
- savedImages: image data records from savedImageStorage.ts.
- sidebarOpen: slide-out menu state.
- themesMenuOpen: app theme menu state.
- activeAppThemeId: root UI theme id from appThemes.ts.
- activeAppThemeBackgroundId: selected theme background variant.
- headerPreviewText: optional large header for preview/export.
- previewText: body text for preview/export.
- homeMode: "compose" or "design".

Derived data
------------
- activeFont: selected from studioData.fonts by activeFontId.
- selectedGlyph: activeFont.glyphs[selectedCharacter], with createEmptyGlyph fallback.
- spacebarGlyph: activeFont.glyphs[" "], with createEmptyGlyph fallback.
- activeCharacters: getVisibleCharacters(activeFont).
- activeAppTheme and activeAppThemeBackground: definitions from src\data\appThemes.ts.

Main responsibilities
---------------------
1. Load project data:
   - Calls loadFontStudioDataWithHealth.
   - Immediately saves migrated data or recovered data through saveFontStudioData.

2. Persist project changes:
   - persist records optional ProjectActivity entries, updates React state, and saves to localStorage.
   - deferProjectSave is used for save-and-next so the UI can advance without waiting on storage.

3. Apply app theme:
   - Writes CSS custom properties onto document.documentElement.
   - Sets root data attributes data-app-theme and data-app-theme-background.
   - Saves app theme ids in localStorage.

4. Route font lifecycle:
   - handleCreateFont uses createFontSet.
   - handleCreatePresetFont creates or selects a FontSet backed by a presetFontId.
   - handleRenameFont, handleDuplicateFont, handleDeleteFont mutate studioData.fonts.

5. Route glyph editing:
   - handleSelectCharacter opens GlyphEditor fullscreen.
   - handleSaveGlyph saves the base glyph while preserving variants.
   - handleSaveGlyphAndNext saves and advances to the next missing glyph when possible.
   - handleSaveGlyphVariant writes or replaces an entry in glyph.variants.

6. Route preview tuning:
   - handleApplyPreviewFontSpacing receives FontSpacingApplyDraft from TextPreview.
   - It applies font-wide metric overrides to non-spacebar glyphs, glyph-specific overrides to specific glyphs, optional guideSettings, and shapeSettings.
   - It records a metrics_batch activity entry.

7. Route saved images:
   - handleSaveImage creates SavedImage records.
   - SavedImagesPanel can share, save, or delete those records.

Child components
----------------
- FontLibrary: current font profile button, profile fullscreen, font creation, preset picker, palette/home/character/writing-style settings, duplicate/export/delete.
- TextPreview: compose home surface and fullscreen image editor/exporter.
- FontMetricsPanel: design-mode glyph queue and missing pair summary.
- GlyphGrid: fullscreen alphabet grid.
- GlyphEditor: fullscreen glyph draw/build editor.
- SavedImagesPanel: fullscreen saved image gallery.

Home modes
----------
homeMode is controlled by FontLibrary:
- compose shows TextPreview.
- design shows the Start designing action and, if enabled in homeSettings, FontMetricsPanel.

Sidebar menu
------------
The menu exposes Alphabet, Saved Images, Design Letters, and Themes. TextPreview renders its preview document/text menu into the sidebar via a portal target with id preview-text-menu-slot.

Planning notes
--------------
- App.tsx is the right place for durable project mutations. Deep feature components should use callbacks passed from App.
- Preview-only state should stay in TextPreview unless the user explicitly applies it to the font.
- Any new persisted field needs fontTypes.ts, fontStorage.ts normalization/defaults, and App mutation plumbing.
- If a feature changes visible characters, selectedCharacter reconciliation in App needs to stay in sync with characterSets.ts.
