What this category covers
-------------------------
This category covers the static web app, PWA install behavior, GitHub Pages publishing, Capacitor Android build/sync, native file save/share, and mobile layout constraints.

Important scripts
-----------------
package.json scripts:
- check:workspace: runs scripts\ensure-d-drive-workspace.mjs.
- dev: Vite dev server on 127.0.0.1.
- build: tsc and vite build.
- android:sync: build then npx cap sync android.
- android:build:debug: android sync then scripts\build-android-debug.mjs.
- preview: vite preview on 127.0.0.1.
- publish:browser: build then copy dist to docs.
- serve:dist: local static dist server.
- verify:renderer: node scripts\verify-renderer.mjs.

Workspace guard
---------------
Most scripts run check:workspace first. The intended workspace is D:\quill. Do not assume an older C-drive copy is correct.

Capacitor config
----------------
capacitor.config.ts:
- appId: com.thecameronboyer.quill.
- appName: Quill.
- webDir: dist.

Native file helpers
-------------------
src\utils\nativeFiles.ts exports:
- isNativeFilePlatform.
- saveNativeFileToDocuments.
- shareNativeFile.

On native platforms:
- saveNativeFileToDocuments writes into Documents / Quill.
- shareNativeFile writes a cache file under share/ and opens Capacitor Share.
- Text data uses UTF8 encoding.
- PNG base64 data is written without text encoding.

Export/share consumers
----------------------
- FontLibrary exports font JSON through native save/share or browser Blob download.
- TextPreview exports preview PNGs through native save/share or browser download.
- SavedImagesPanel shares and saves saved PNG data URLs.

PWA
---
src\pwa.ts registers sw.js only in production and only when serviceWorker is available.

public\manifest.webmanifest:
- name: Quill.
- short_name: Quill.
- standalone display.
- portrait orientation.
- icons: icon-192.png, icon-512.png, icon.svg.

public\sw.js:
- CACHE_NAME quill-v42.
- caches app base, manifest, and icons at install.
- removes old caches at activate.
- fetch handler uses network-first with cache fallback for same-origin GET requests.

Android
-------
The android folder is a Capacitor Android project. Build notes point to docs\android-app.md. APK artifacts are present in docs\downloads for prior versions.

Mobile layout
-------------
styles.css includes many mobile-specific sections:
- safe-area padding with env(safe-area-inset-*).
- body.editor-fullscreen-open to prevent background scroll.
- phone-image-fullscreen bottom panel sizing.
- fullscreen draw/build canvas height calculations.
- tablet portrait coarse-pointer media rules.
- Codex/mobile preview mode comments around always showing the phone layout.

Planning notes
--------------
- Source-pack refreshes do not need npm run build.
- App source changes that affect mobile/export should usually run npm run build, and sometimes android:sync or android:build:debug if native behavior changes.
- For local browser verification, prefer the Vite dev server or serve:dist depending on what changed.
- Native file changes need browser fallback and Capacitor fallback checks.
- PWA cache version should change when public cached assets need a forced refresh.
