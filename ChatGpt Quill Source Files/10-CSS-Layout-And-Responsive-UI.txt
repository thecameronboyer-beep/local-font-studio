What this category covers
-------------------------
This category covers the global app layout, responsive rules, fullscreen overlays, phone preview editor, glyph editor workbench, construction workbench, theme CSS variables, and layout hazards.

Global structure
----------------
App.tsx renders:
- main.app-shell.
- menu-toggle.
- sidebar-backdrop.
- aside.sidebar-menu.
- header.app-header.
- div.workspace.
- div.left-stack.
- fullscreen overlays for grid, saved images, and glyph editor.

styles.css is large and source-order sensitive. Later rules refine or override earlier component sections. Check the bottom of the file before assuming an earlier rule is final.

Theme variables
---------------
App themes from appThemes.ts write CSS variables such as:
- --app-bg.
- --app-bg-gradient.
- --app-surface.
- --app-surface-raised.
- --app-surface-solid.
- --app-text.
- --app-heading.
- --app-muted.
- --app-primary.
- --app-accent.
- --app-border.
- --app-shadow.

Many later component rules use these variables. If a palette or theme looks wrong, check root CSS variables first.

Fullscreen body lock
--------------------
App toggles body.editor-fullscreen-open when editor, alphabet grid, saved images, or preview image fullscreen is open. CSS hides menu/sidebar elements and prevents unwanted body scroll in fullscreen contexts.

Responsive rules
----------------
styles.css includes:
- @media (max-width: 980px).
- @media (max-width: 640px).
- @media (max-width: 560px).
- @media (min-width: 700px) and (max-width: 1024px) and (orientation: portrait) and (hover: none) and (pointer: coarse).

There are also Codex/mobile preview mode comments and rules that force phone-like layout in wide browser panes.

Layout hazards
--------------
- Many buttons and controls are fixed-height to protect the preview surface.
- Safe-area padding is used in fullscreen top and bottom bars.
- Long-skinny preview has special CSS even though it is no longer a normal export preset button.
- transparent-preview changes background visuals for live inspection.
- CSS :has() is used in some fullscreen settings areas.
- Source order matters. Later theme/layout blocks can override earlier component declarations.

Planning notes
--------------
- For preview layout changes, inspect TextPreview markup and the phone-image CSS together.
- For glyph editor changes, check draw fullscreen, construction fullscreen, and regular editor-panel states.
- For font profile changes, check both home compact profile and fullscreen profile menu.
- After source changes, use browser screenshots in at least a narrow mobile-ish viewport and a desktop viewport if the UI is visible.
