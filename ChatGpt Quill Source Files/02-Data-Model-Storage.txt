Data Model and Storage
======================

Main files
----------
- src\types\fontTypes.ts
- src\storage\fontStorage.ts
- src\storage\savedImageStorage.ts
- src\data\characterSets.ts
- src\components\TextPreview.tsx

What this category covers
-------------------------
This category covers the durable data model, localStorage keys, defaults, schema normalization, migrations, backups, project import/export, font import/export, saved image storage, and preview document storage.

Project data
------------
FontStudioData:
- version: currently 3.
- activeFontId: selected font id.
- fonts: FontSet[].
- activityLog: capped recent project events.
- lastBackupAt: optional timestamp.

FontSet:
- id, name, createdAt, updatedAt.
- glyphs: Record<string, Glyph>.
- characterSettings: optional character groups.
- guideSettings: ascender, xHeight, baseline, descender, leftBound, rightBound.
- homeSettings: visible home sections.
- writingStyleSettings: enabled draw/build styles.
- shapeSettings: font-wide heightScale, widthScale, letterSpacing.
- theme: font palette/background/ink/accent/texture settings.
- renderProfile: optional "quillParchment".
- presetFontId: optional preset source id.

Glyph data
----------
Glyph:
- character.
- strokes: freehand GlyphStroke[].
- construction: optional build-mode GlyphConstruction.
- decorations: glyph-level GlyphDecoration[].
- variants: alternate glyph drawings.
- width, height, xAdvance, baselineOffset, leftBearing, rightBearing.
- updatedAt.

GlyphStroke:
- id.
- points with normalized x/y, optional pressure, ink, spread.
- size normalized to glyph canvas.
- optional color and highlightColor.
- strokeTool: "pen" or "quill".
- inkEffect: "none", "dramaticPooling", "subtleSpread", or "bubbleHighlight".

GlyphConstruction:
- paths made of ConstructionAnchorPoint records.
- supports line and curve segments.
- supports corner, smooth, symmetric, rounded point types.
- supports sharp, rounded, and chamfered corner styles.
- paths can be open/closed and filled/stroked.

GlyphDecoration:
- currently kind "googly-eyes".
- expression can be googly, happy, angry, sad, tired, or stoned.
- x, y, size are normalized.

Character sets
--------------
src\data\characterSets.ts defines:
- uppercase A-Z.
- lowercase a-z.
- header letters using keys like "header:A".
- numbers 0-9.
- punctuation . , ? ! : ; ' " - ( )
- forgotten: currently an empty array.
- spacebar: " ".

getVisibleCharacters(font) always includes the standard supported characters. It adds header letters when showHeaderLetters is true, forgotten characters when showForgotten is true, and spacebar when showSpacebar is true.

Storage keys
------------
- quill:data:v1: main project data.
- quill:backups:v1: project backups.
- quill:data:corrupt:v1: corrupt payload preservation.
- quill:saved-images:v1: saved PNG data URLs.
- quill:app-theme:v3: UI theme id.
- quill:app-theme-background:v1: UI theme background id.
- quill:preview-documents:v1: preview documents saved by TextPreview.

fontStorage.ts behavior
-----------------------
- CURRENT_STORAGE_VERSION is 3.
- createFontSet creates all glyph records for fontCharacters and gives non-spacebar glyphs the font baseline.
- loadFontStudioDataWithHealth returns data plus StorageHealthCheck.
- migrateFontStudioData normalizes old/malformed data into schema v3.
- normalizeFont filters removed old data, fixes duplicate ids, normalizes theme, character settings, home settings, guide settings, writing styles, shape settings, and preset ids.
- normalizeGlyph fills missing glyphs and normalizes strokes, variants, construction, decorations, and metrics.
- normalizeConstruction drops old construction sample paths with ids starting construction_sample_.
- saveFontStudioData creates automatic backups unless createBackup is false.
- importFontStudioProject accepts either the full export wrapper or raw data.
- exportFontStudioProject and exportFontSet write schemaVersion 3 JSON.

Backups and activity
--------------------
- MAX_ACTIVITY_ENTRIES is 80.
- MAX_BACKUPS is 12.
- AUTO_BACKUP_INTERVAL_MS is 5 minutes.
- Activity types include backup, font_create, font_delete, font_duplicate, font_rename, glyph_edit, import, export, metrics_batch, migration, recovery, restore.
- If current project storage is malformed, the corrupt data is copied to quill:data:corrupt:v1 and the newest backup is restored when possible.

Saved images
------------
src\storage\savedImageStorage.ts stores up to 24 images. A SavedImage includes createdAt, fontName, height, id, imageDataUrl, message, and width.

Preview documents
-----------------
Preview documents are defined inside TextPreview, not in fontTypes.ts. They are preview-only records with id, name, text, headerText, settings, textLayers, textTargetPlacements, and updatedAt. They are stored separately from the main project under quill:preview-documents:v1.

Planning notes
--------------
- Durable changes require type, default, normalization, migration, and App wiring updates.
- Preview-only changes can usually stay in TextPreview state and preview document storage.
- Metric changes affect glyphRenderer.ts, TextPreview measurement, preset measurement, hit targets, SpacingControls, FontMetricsPanel, and apply-to-font behavior.
- If changing character availability, update characterSets.ts, storage glyph creation, visible character reconciliation, grid, metrics panel, and preview header-letter helpers.
