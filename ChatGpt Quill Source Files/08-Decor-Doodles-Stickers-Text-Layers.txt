What this category covers
-------------------------
This category covers preview-page decoration layers, preview doodles, preview stickers, ornaments, text layers, selection/move modes, glyph-level eye stickers, and the export interactions for all of those layers.

Two different decor scopes
--------------------------
1. Glyph-level decorations:
   - Stored in Glyph.decorations.
   - Edited in GlyphEditor/GlyphCanvas.
   - Rendered by glyphRenderer.drawGlyphDecoration as part of a glyph.
   - Persisted with the font project.

2. Preview-page decorations:
   - Stored in TextPreview state and preview documents.
   - Include doodles, stickers, ornaments, and text layers.
   - Rendered into preview/export canvas.
   - Not stored in FontSet unless converted through a future feature.

Preview style state
-------------------
TextPreview owns:
- previewDoodles.
- previewStickers.
- previewTextLayers.
- previewTextTargetPlacements.
- hiddenPreviewTextTargetIds.
- selectedPreviewDoodleId.
- selectedPreviewStickerId.
- selectedPreviewTextLayerId.
- styleDrawMode.
- styleSelectModeActive.
- styleMoveModeActive.
- styleSelectTarget.
- style doodle brush, tool, smoothing, and ink effect.
- sticker edit panels for sleepiness, redness, face mood, size, width, and height.

Style drawers
-------------
StyleDrawer values:
- doodle.
- ornaments.
- select.
- stickers.
- text.

The fullscreen Add category opens these drawers or popovers. CSS for this area is under phone-style, style-control, style-select, style-sticker, and style-text-layer classes.

Preview doodles
---------------
Preview doodles are freehand strokes drawn directly on the preview/export canvas. They have:
- id.
- points.
- color.
- size.
- tool: pen, quill, or line.
- smoothing.
- optional inkEffect.

TextPreview has separate pointer handlers for the style canvas and viewer canvas. Doodle rendering uses drawPreviewDoodles and getPreviewDoodleRenderPoints.

Preview stickers
----------------
PreviewStickerKind:
- eyes.
- strawberry-vine-divider.
- lace-ribbon-divider.
- botanical-branch-divider.

Eyes are custom canvas-rendered. The other three are image assets under:
public\assets\stickers\strawberry-market

TextPreview preloads image sticker assets and draws them in drawPreviewImageSticker. It supports sticker move, selection, delete, resizing, width/height adjustment, and face settings for eye stickers.

Ornaments
---------
Ornaments are the non-eye sticker assets. They use the same PreviewSticker model but are opened through the ornaments drawer. They are drawn after text content as page decor.

Text layers
-----------
PreviewTextLayer:
- id.
- text.
- fontId.
- x, y.
- sizeScale.
- color.
- alignment.

Text layers can use:
- any custom FontSet available in the project.
- any bundled preset font option.

Text layer font picker
----------------------
TextPreview builds availableTextFontOptions from:
- availableFonts, including previewFont with pending metric overrides.
- bundled fontPresets.

PreviewTextFontNamePreview draws custom font options with drawGlyph and preset options with getFontPresetCanvasFont.

Text target placement
---------------------
TextPreview treats header, body, and text layers as selectable text targets. previewTextTargetPlacements stores normalized x/y placement by target id. hiddenPreviewTextTargetIds can hide header/body/layers for document save/export.

Selection modes
---------------
StyleSelectTarget:
- text.
- letter.
- stickers.
- ornaments.
- doodles.

StyleMoveTarget:
- text.
- stickers.
- ornaments.
- doodles.

The fullscreen preview can show selection outlines for active/available targets and then move or edit the selected layer.

Render order
------------
The preview export order is:
1. background.
2. long-skinny paper rules when applicable.
3. header/body text and text layers with effects.
4. decor layers: doodles, stickers, ornaments.
5. selection outlines only for editor UI, not final export.

Glyph-level eye stickers
------------------------
GlyphCanvas supports the eyes tool and draggable eye sticker requests. Glyph-level eye expressions render through drawGlyphDecoration and are saved in the glyph or variant.

Preview documents
-----------------
Saved preview documents currently persist textLayers and textTargetPlacements. They do not persist previewDoodles or previewStickers in the loaded document model at refresh time, even though the UI has preview-page doodle/sticker state. If future planning expects saved decor documents, verify and extend the PreviewDocument model first.

Planning notes
--------------
- Keep glyph-level decorations and preview-page decorations distinct.
- New preview decor that should survive document load needs PreviewDocument fields plus load/save normalization.
- New glyph decoration kinds need fontTypes.ts, fontStorage.ts normalization, glyphRenderer.ts drawing, GlyphEditor/GlyphCanvas editing, and export checks.
- Text layer rendering must respect custom and preset font branches, text effects, selection outlines, and hidden target state.
