What this category covers
-------------------------
This category covers fullscreen glyph editing, freehand design mode, build/construction mode, variants, undo/redo, ink tools, stickers, guides, nudge/center controls, per-glyph metrics, and save flows.

GlyphEditor responsibilities
----------------------------
GlyphEditor owns the editable draft state for one glyph and passes low-level drawing state into GlyphCanvas or GlyphConstructionCanvas. It also renders the toolbars, drawers, live preview, variation controls, and save actions.

Important editor state
----------------------
- draftGlyph: cloned editable glyph.
- activeVariation: "base", "new", or a variant index.
- editorMode: "draw" or "construction".
- tool: pen, quill, line, eraser, eyes, select, or pan.
- constructionTool: select, point, or delete.
- brushSize and construction thickness.
- inkColor, highlightColor, inkEffect.
- smoothingMode: raw, gentle, strong.
- eraserMode: stroke or point.
- selectMode: moveStroke, editPoint, smoothCircle, spreadCircle.
- viewScale and viewOffset.
- showGuides.
- selectedStrokeId, selectedDecorationId, construction selection.
- history stacks for undo/redo.
- ink preset slots and construction ink preset slots.
- fullscreen drawer state for ink, background, guides, more, construction add/ink/paths/settings/more.

Writing-style constraints
-------------------------
The active font's writingStyleSettings controls whether draw and build modes are available. FontLibrary prevents disabling both modes. GlyphEditor reconciles editorMode when one mode is unavailable and hides the mode switch when only one style is enabled.

Draw mode
---------
GlyphCanvas handles the actual pointer interaction on a responsive canvas. It supports:
- pen and quill strokes.
- straight line strokes.
- eraser by whole stroke or by point/segment.
- eye sticker placement and movement.
- selection by stroke, point, smooth circle, or spread circle.
- pan and zoom.
- guide editing when guideEditMode is enabled.
- optional reference glyph overlay.
- Pixi overlay for heavy ink effects.

Glyph stroke data
-----------------
Strokes use normalized coordinates. Pointer pressure is captured when available. Stroke size is normalized against the rendered canvas basis. The active stroke stores:
- color.
- highlightColor for bubbleHighlight.
- inkEffect.
- size.
- strokeTool.
- points.

Ink effects
-----------
Draw mode currently supports:
- none.
- subtleSpread.
- dramaticPooling.
- bubbleHighlight.

subtleSpread can be generated by slow pointer movement or the spread-circle select tool. dramaticPooling adds stronger edge pooling. bubbleHighlight adds a highlight pass.

Sticker decorations
-------------------
Glyph-level stickers currently use GlyphDecoration kind "googly-eyes". Expression options include googly, happy, angry, sad, tired, and stoned. Decorations are part of a glyph and are saved with the glyph or variant.

Build mode
----------
GlyphConstructionCanvas edits GlyphConstruction paths. It supports:
- adding anchors.
- selecting and moving points, handles, or whole paths.
- deleting points/paths.
- snapping to grid, guides, centerline, and anchors.
- open/closed paths.
- line or curve segments.
- sharp, rounded, and chamfered corners.
- filled and stroked paths.

Construction renderer
---------------------
Build-mode paths are rendered in src\render\constructionRenderer.ts. Construction shapes are drawn inside drawGlyph before freehand strokes and decorations, so build-mode letters and drawn strokes can coexist.

Guide editing
-------------
Guide settings are shared by FontLibrary, GlyphEditor, GlyphCanvas, and TextPreview. The guide keys are:
- ascender.
- xHeight.
- baseline.
- descender.
- leftBound.
- rightBound.

fontGuides.ts clamps guide values so vertical zones stay ordered and left/right bounds keep a minimum distance.

Metrics and spacing
-------------------
SpacingControls edits glyph metrics:
- width.
- leftBearing.
- rightBearing.
- xAdvance.
- baselineOffset.

GlyphEditor also has nudge and center controls that translate strokes, decorations, and construction points together.

Variants
--------
A glyph can have variants. GlyphEditor can:
- edit the base glyph.
- start a new empty variation from the existing glyph.
- edit an existing variation.
- save the current draft as a new variation.
- update an existing variation.

Rendering picks variants through selectPreviewGlyph using a deterministic seed so repeated letters can vary.

Save flows
----------
- Save base calls onSaveGlyph.
- Save variation calls onSaveGlyphVariant.
- Save and next calls onSaveGlyphAndNext, then App advances to the next missing visible glyph when possible.
- App merges saved base glyphs with existing variants so variant arrays are not lost.

Planning notes
--------------
- GlyphEditor is large. Keep changes scoped to the relevant mode or drawer.
- A glyph edit should update draft state first, then save through App callbacks.
- Any new glyph data must be cloned, normalized, exported, imported, and rendered.
- If adding tools that affect preview/export, update glyphRenderer.ts and TextPreview canvas paths too.
- Fullscreen draw/build CSS has specialized no-scroll workbench rules; see 10-CSS-Layout-And-Responsive-UI.txt.
