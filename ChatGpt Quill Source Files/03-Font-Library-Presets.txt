What this category covers
-------------------------
This category covers the current font profile UI, font creation, preset fonts, font switching, rename, duplicate, delete, JSON export, palette selection, character settings, home-screen settings, writing-style settings, and guide setup.

FontLibrary shape
-----------------
FontLibrary receives all font mutations as callbacks from App. It does not own the durable project. It renders:
- the active font preview button on the home screen.
- the fullscreen font profile/settings surface.
- the font switcher list.
- create new font form.
- preset picker.
- palette selector.
- writing-style toggles.
- home-screen section toggles.
- character options and design-line editor.
- duplicate, export JSON, delete actions.
- home mode toggle for Design and Compose.

Active profile row
------------------
The visible home surface is intentionally compact. It shows only the active font preview row, then the Design/Compose toggle and any mode-specific controls.

FontNamePreview
---------------
FontNamePreview draws the font name to a canvas. It handles:
- compact and full variants.
- themed row backgrounds.
- preset font names through getFontPresetCanvasFont.
- custom font names through drawGlyph and selectPreviewGlyph.
- ResizeObserver redraw.
- document.fonts load wait for preset fonts.

Font creation
-------------
Creating a font uses:
- name input.
- profile selector: Pen or Quill.
- optional Header Letters.
- optional Space Bar.
- design-line setup before creation.

App.handleCreateFont calls createFontSet(name, renderProfile, characterSettings, guideSettings, shapeSettings). Quill profile defaults to the quillParchment theme. Plain fonts default to getDefaultFontPaletteTheme.

Preset-backed FontSet behavior
------------------------------
App.handleCreatePresetFont creates a normal FontSet plus presetFontId. It reuses an existing preset font if one already exists for that preset id. Preset FontSets hide drawActions and glyphQueue by default in homeSettings, but still keep glyph metrics so preview tuning can affect preset rendering.

Important current behavior: TextPreview applies width, height, baseline, bearings, xAdvance, spacebar, and shape settings to preset-font measurement/rendering through getPresetCharacterMetrics and drawPresetTextToCanvas. Preset tuning is not only visual fallback text.

Palette selector
----------------
FontLibrary uses fontPalettes from src\data\palettes.ts. Current palette ids:
- strawberryMarket
- midnightCorrespondence

Changing a palette writes font.theme through onUpdateFontSettings. getDefaultFontPaletteTheme picks the palette's default background and default texture.

Writing style settings
----------------------
FontLibrary renders draw/build checkboxes from fontWritingStyleOptions:
- draw: label Design.
- build: label Build.

updateActiveWritingStyle refuses to turn off both styles. GlyphEditor also reconciles unavailable editor modes.

Home screen settings
--------------------
fontHomeSectionOptions:
- drawActions: Design actions.
- exportControls: Export controls.
- previewText: Preview text.
- glyphQueue: Glyph queue.

These flags decide which home sections appear. App passes activeFont.homeSettings.visibleSections into TextPreview and checks glyphQueue before showing FontMetricsPanel in design mode.

Character settings
------------------
FontLibrary exposes:
- Header Letters.
- Space Bar.

showForgotten exists in the data model but is not currently exposed in this UI, and forgotten is currently empty in characterSets.ts.

Design-line editor
------------------
FontGuideEditor is embedded in FontLibrary for new fonts and active fonts. It draws a 720px canvas with:
- ascender.
- x-height.
- baseline.
- descender.
- left bound.
- right bound.

It uses fontGuideRows and clampFontGuideSettings from src\utils\fontGuides.ts to keep vertical lines ordered and horizontal bounds valid.

Export JSON
-----------
downloadFontJson uses exportFontSet. In Capacitor it saves and shares through nativeFiles.ts. In the browser it creates a Blob and clicks a download link.

Planning notes
--------------
- FontLibrary should stay a UI/control surface. Durable mutations should stay in App callbacks.
- Preset fonts are real FontSets with metrics, home settings, theme, and a presetFontId. Do not treat them as only static CSS fonts.
- When adding a font-level setting, update fontTypes.ts, fontStorage.ts defaults/normalization, FontLibrary UI, and the render path that consumes it.
- If changing profile/fullscreen layout, check 10-CSS-Layout-And-Responsive-UI.txt for source-order and safe-area rules.
