What this category covers
-------------------------
This category covers app UI themes, font palettes, font themes, preview backgrounds, editor backgrounds, textures, manuscript page rendering, and CSS variables.

App themes
----------
src\data\appThemes.ts defines AppThemeId:
- classic.
- midnightCorrespondence.
- strawberryMarket.

App themes are UI-level themes. App.tsx stores them in localStorage:
- local-font-studio:app-theme:v3.
- local-font-studio:app-theme-background:v1.

App.tsx writes the chosen theme's cssVariables onto document.documentElement and sets:
- data-app-theme.
- data-app-theme-background.

classic
-------
The original dark studio look. It has no background variants.

midnightCorrespondence
----------------------
Light literary stationery theme with three background variants:
- parchmentIvory.
- moonlitBlue.
- dustySlate.

strawberryMarket
----------------
Cream paper, berry controls, cocoa ink, and leaf-green active states. It uses the strawberryMarket font palette values.

Font palettes
-------------
src\data\palettes.ts defines FontPaletteId:
- strawberryMarket.
- midnightCorrespondence.

Font palettes are font/export-level palettes. They provide:
- main swatches.
- accent swatches.
- ink swatch.
- background presets.
- default background id.
- default texture.

getDefaultFontPaletteTheme returns a FontTheme:
- accentColor.
- backgroundColor.
- backgroundStyle.
- backgroundTexture.
- inkColor.
- paletteId.

FontTheme model
---------------
FontTheme is stored on FontSet and used by:
- FontLibrary previews/profile rows.
- GlyphEditor canvas/background controls.
- GlyphCanvas and GlyphConstructionCanvas backgrounds.
- TextPreview default image settings.
- TextPreview export backgrounds and ink color.

FontLibrary palette selector
----------------------------
The profile fullscreen shows a Palette section. Selecting a palette calls getDefaultFontPaletteTheme(palette.id) and writes font.theme through App.onUpdateFontSettings.

Glyph editor backgrounds
------------------------
GlyphEditor has editor background presets and texture presets. It can update the active font theme through onUpdateFontTheme. GlyphCanvas draws:
- solid/paper-like fills.
- themed backgrounds.
- grain, fiber, canvas, and woven textures.
- guide lines and board edges over the background.

GlyphConstructionCanvas backgrounds
-----------------------------------
Build mode has its own getThemeBackground helper and draws construction paths against the active theme colors. It shares guide settings with the rest of the editor.

TextPreview backgrounds
-----------------------
TextPreview has backgroundPresets that include palette backgrounds plus:
- paper.
- parchment.
- manuscript.
- midnight.
- rage.
- blush.
- sage.
- sky.
- lavender.
- lined.
- grid.

TextPreview can also toggle transparency. Transparent export skips normal page background and uses transparent-preview CSS checker styling for the live preview.

Manuscript background
---------------------
The manuscript background uses:
- public\assets\parchment-clean-vellum.png.
- MANUSCRIPT_PARCHMENT_SRC in TextPreview.
- manuscriptAge, manuscriptEdges, manuscriptFibers, manuscriptInkSoak, manuscriptRuling, manuscriptStains sliders.

drawManuscriptPage draws the parchment layer, fibers, stains, edges, ruling, and content ink integration.

CSS variables
-------------
styles.css relies heavily on CSS variables:
- --app-bg, --app-bg-gradient.
- --app-surface, --app-surface-raised, --app-surface-solid.
- --app-text, --app-heading, --app-muted.
- --app-primary, --app-accent, --app-gold.
- --app-border, --app-shadow.

Later style blocks override or normalize many surfaces to these variables. Check source order before changing colors.

Planning notes
--------------
- App themes and font themes are separate. App themes affect UI chrome; FontTheme affects glyph editor and exported preview look.
- If adding a new font palette, update palettes.ts, any preview background options, FontLibrary selector behavior, and storage normalization for paletteId.
- If adding a new BackgroundStyle or BackgroundTexture, update fontTypes.ts, fontStorage.ts normalization, TextPreview background rendering, GlyphCanvas, GlyphConstructionCanvas if needed, and CSS preview classes if needed.
- Manuscript changes should be tested in live preview and exported PNG because the offscreen ink compositing path can differ from normal backgrounds.
