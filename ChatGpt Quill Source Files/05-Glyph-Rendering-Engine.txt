Glyph Rendering Engine
======================

Main files
----------
- src\render\glyphRenderer.ts
- src\render\constructionRenderer.ts
- src\render\pixiInkRenderer.ts
- src\components\TextPreview.tsx

What this category covers
-------------------------
This category covers how glyphs are selected, measured, drawn, hit-tested, and shared across editor, preview, font list, alphabet grid, and export.

Core renderer responsibilities
------------------------------
src\render\glyphRenderer.ts exports:
- hasDrawnGlyph.
- getDrawableGlyphVariants.
- findPreviewGlyph.
- selectPreviewGlyph.
- getFontHeightScale.
- getFontWidthScale.
- getGlyphRenderScales.
- getGlyphLeftBearingOffset.
- getGlyphTopForBaseline.
- getGlyphAdvance.
- getSpacebarAdvance.
- drawStrokePath.
- drawGlyphDecoration.
- drawGlyph.

Glyph selection
---------------
findPreviewGlyph:
- returns the first drawable exact glyph or variant.
- falls back between lowercase and uppercase when needed.

selectPreviewGlyph:
- returns a deterministic variant based on a seed.
- falls back between case variants.
- is used in preview, font name previews, and repeated text rendering so the same character can vary across occurrences.

Drawn state
-----------
hasDrawnGlyph returns true if:
- the base glyph has construction marks, decorations, or strokes.
- any variant has construction marks, decorations, or strokes.

FontMetricsPanel uses this for missing groups. GlyphGrid uses it to show completion and variants.

Metrics
-------
Glyph metrics are central to every renderer:
- width and height scale the glyph visual bounds.
- xAdvance controls cursor advance.
- baselineOffset controls vertical placement relative to baseline.
- leftBearing and rightBearing affect offset and advance.
- shapeSettings.widthScale and shapeSettings.heightScale apply font-wide scaling.
- shapeSettings.letterSpacing applies extra tracked advance in TextPreview.

getGlyphAdvance uses the larger of xAdvance and leftBearing + width + rightBearing, multiplied by font width scale. It clamps to a minimum based on font size.

Baseline math
-------------
getGlyphTopForBaseline computes glyph y from baselineY, baselineOffset, fontSize, and height scale. TextPreview uses this in both custom glyph drawing and preset fallback drawing.

Custom glyph drawing
--------------------
drawGlyph:
1. draws build-mode construction paths.
2. draws all strokes through drawStrokePath.
3. draws glyph decorations.

It accepts renderProfile and backgroundTexture so quill strokes and ink spread can adapt to the font/background style.

Stroke drawing
--------------
drawStrokePath supports:
- pen-like pressure-width strokes.
- quill strokes with nib angle and texture.
- subtle spread marks.
- dramatic pooling.
- bubble highlight.
- fallback color.
- rectangular scaling for width/height glyph metrics.

Quill stroke rendering uses a 38 degree nib angle and seeded texture so marks are stable between renders.

Construction drawing
--------------------
src\render\constructionRenderer.ts supports:
- line and Bezier curve segments.
- rounded and chamfered corners.
- open and closed paths.
- filled paths.
- per-path stroke width and color fallback.

Construction paths are scaled by glyph width/height in drawGlyph.

Pixi ink renderer
-----------------
src\render\pixiInkRenderer.ts mirrors heavier ink effects in Pixi for the live GlyphCanvas overlay. It exports:
- hasPixiInkEffect.
- renderPixiInkLayer.

GlyphCanvas uses the Pixi layer for strokes with heavy effects, then skips drawing those same effects in the main canvas when appropriate. This keeps interactive drawing responsive.

Preset font rendering path
--------------------------
Preset fonts do not use drawGlyph for letter shapes. TextPreview has a parallel preset path:
- setPresetCanvasFont.
- getPresetAdvanceScale.
- getPresetCharacterMetrics.
- measurePresetTextRun.
- buildPresetWordWrappedLines.
- buildPresetCharacterWrappedLines.
- drawPresetTextToCanvas.

Current important behavior: preset text honors font metrics from the backing FontSet. The preset path applies widthScale, heightScale, baselineOffset, left/right bearings, xAdvance, spacebar advance, and font-wide shape settings before drawing canvas text.

Preview hit targets
-------------------
TextPreview creates hit targets for letters so clicking a preview letter can select or tune it. Custom glyphs and preset-font characters both need hit targets to stay aligned with the actual drawn text.

Renderer consumers
------------------
- TextPreview: full preview/export image, hit targets, metric tuning, text layers.
- GlyphCanvas: editor live canvas.
- GlyphGrid: alphabet mini previews.
- FontLibrary: active font and switcher previews.
- FontMetricsPanel: missing glyph and missing pair detection.
- SavedImagesPanel: displays already-rendered PNGs, not live glyphs.

Planning notes
--------------
- Any metric change must be checked in both custom glyph and preset font paths.
- If drawGlyph changes, verify TextPreview export, GlyphCanvas editor, GlyphGrid cells, and FontLibrary previews.
- If adding a new ink effect, update fontTypes.ts, fontStorage.ts normalization, GlyphEditor controls, glyphRenderer.ts, and pixiInkRenderer.ts if the editor needs the effect during live drawing.
- If adding a new construction feature, update constructionRenderer.ts, GlyphConstructionCanvas, fontStorage.ts normalization, and cloning helpers in GlyphEditor.
